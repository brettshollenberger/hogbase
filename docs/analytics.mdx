# Analytics

## Overview

The analytics module provides a comprehensive integration with PostHog for tracking user behavior, experiments, and important events in your application.

## Configuration

Analytics is configured using environment variables:

```typescript
VITE_POSTHOG_KEY=your-project-api-key
```

The PostHog client is automatically initialized in browser environments with:
- API Host: https://us.i.posthog.com
- UI Host: https://us.posthog.com

## Core Functions

### User Identification

```typescript
initializeAnalytics(user?: { id?: string; email?: string; name?: string; [key: string]: any })
identify(userId?: string, traits?: Record<string, any>)
```

Identifies users in PostHog. The `initializeAnalytics` function should be called on your index page.

### Page & Event Tracking

```typescript
trackPageView(pageName: string, section?: string)
trackEvent(eventName: string, properties?: Record<string, any>)
```

Track page views and custom events.

### User Interaction Tracking

```typescript
trackLinkClick(linkText: string, section?: string)
trackCTAClick(buttonText: string, section?: string)
```

Track user interactions with links and CTAs.

### Pricing & Signup Tracking

```typescript
trackTierSelection(tier: { id: string; name: string; price: number })
trackSignupAttempt(tier: { id: string; name: string; price: number })
trackSignupSuccess(tier: { id: string; name: string; price: number; email: string })
trackSignupError(error: string, tier?: { id: string; name: string })
```

Track the entire pricing and signup flow.

### Experiment Tracking

```typescript
trackExperimentView(posthogClient: any, experiment: string, variant: string, source: string = 'url')
```

Track when users view different experiment variants.
